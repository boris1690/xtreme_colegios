1. General

Dropper is a skin for Bootstrap, wich means that can be used with any site built with the Bootstrap framework. With easy installation and customization thanks to the LESS files that are included. LESS is a tool to quickly declare CSS rules and after compile the .less files, get the real .css file ready to include in your site. Dropper has many improvements to the original Bootstrap styles based on latest CSS3 functionalities. A modern browser to test it is highly recommended.

Following explains the main files

  • bootstrap.min.css: contains the main styles of the skin. There are five files for each theme included. Fire, Water, Wind, Earth and Love. Each one represents a different color according to his name. The only difference between each theme is the color, other rules like border, margin, paddings, etc, are the same for each one. This file you must include into your Bootstrap based site to apply the skin. See Usage section to know how to do it.
  • bootstrap.min.js: contains the main plugins used across the skin.
  • docs.css: contains the main styles for the demo pages. You can use this file if you want to know how works some components of the demo page or include some of them into your site. For example the big cute button of the home page. This file is located in demo/assets/css/
  • less: this is not a file, is a folder where are located LESS files modified by the Dropper skin. If you need to customize Dropper, you must edit this files and then compile with some tool that you can find here

    Another option to customize this skin online is using the Theme Roller App designed exclusively for buyers. This tools is based on the standar Boostrap customize page and can be found HERE

2. File structure

Within the download you'll find the following file structure and contents

Once downloaded, unzip the compressed zip file to see the structure. You'll see something like this:

  dropper/
  ├── assets/
  │   ├── css/
  │   │   ├── bootstrap.[theme].min.css
  │   │   ├── bootstrap.min.css
  │   │   ├── bootstrap-responsive.css
  │   │   └── bootstrap-responsive.min.css
  │   ├── font/
  │   ├── img/
  │   └── js/
  │       ├── bootstrap.min.js
  │       └── bootstrap.js
  ├── less/
  │   └── *.less
  └── *.html

Each theme is a bootstrap file named with the theme name and the styles and rules for that theme. There is also the unminified version for each css file boostrap.[theme].css

If you will use the reponsive features don't forget to include the file bootstrap-responsive.min.css located under the css/ folder, this file is common for all themes.

bootstrap.min.css is the default theme used in demo, which is blue

3. Usage

Installation

Replace style

  • Go to css/ folder
  • Copy the css file of your choosen theme
  • Paste and replace the file into your style folder of your bootstrap based site.

New installation

  • Go to css/ folder
  • Copy the css file of your choosen theme
  • Paste it into your style folder and rename it to bootstrap.min.css.
  • Link to the file in all your pages with the next code (add it between the <head>):

    <link href="css/bootstrap.min.css" rel="stylesheet">
  • Make sure to fix the path to the stylesheet if your project uses some different.

Plugin installation

  • Just copy the bootstrap.min.js to your JavaScript folder
  • Include at the bottom of your pages as follows: 
    <script src="js/bootstrap.min.js"></script>
  • Make sure to fix the path to the script if your project uses some different.

Font installation

If you want to include the font used in demo pages, add the following code between the <head> of your pages

<link href='http://fonts.googleapis.com/css?family=PT+Sans:400,700,400italic' rel='stylesheet' type='text/css'>

FontAwesome on IE7

To support FontAwesome on IE7 you need to add the next lines to your html files, between the head tags

  <!--[if lte IE 7]>
  <link href="assets/css/font-awesome-ie7.min.css" rel="stylesheet">
  <![endif]-->

With this steps done, you are ready to run your site with the a new Bootstrap style. Hope you Enjoy It! :)

6. Configuring the Twitter API v1.1

Twitter widget is build on Twitter API 1.1. For the new Twitter API you need to have API keys (similar to Facebook). To get the access keys you need to sign in and create an application on the Twitter Developer Site. Once your application is created Twitter will generate your access keys and tokens.

The following process requires a Twitter accound. If you don’t already have a Twitter account, you can sign up at the Twitter home page to create one.

  • First you need to go to the Twitter Developer Site. Go to https://dev.twitter.com/
  • Login with your Twitter account and go to My Applications.
  • From there click on the “Create a new application” button (optionally go to this new application link.
  • Fill the form with the information required.
  • Once you’re done, a new page will be displayed, and from there you’ll need the following
    • Consumer Key
    • Consumer Secret
    • Access Token
    • Access Token Secret
  • Now go the folder assets/php/ and open the file twitter.config.php and fill in with the information obtained before

Important Twitter API v1.1 requires PHP so, in order to work you need to run your site from a local or web server. Otherwise, PHP files will not be processed.
